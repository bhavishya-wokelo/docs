## Quick start

The Company Monitoring API fetches the latest news articles for a given company, enriched with AI-generated summaries, event categories, publisher metadata, and article text.

Results are returned synchronously in the response

`GET: /api/enterprise/company/news/`

NOTE: This is a synchronous API, which means the results will be returned in the HTTP response itself.

Lets start with a simple one.

<CodeGroup>

```json cURL expandable wrap
curl --location 'https://api.wokelo.ai/api/enterprise/company/news/?company=tesla' \
--header 'Authorization: Bearer Token' \
--header 'Content-Type: application/json'
```

```json Python expandable wrap
import requests
import json

url = "https://api.wokelo.ai/api/enterprise/company/news/?company=tesla"

payload = {}
headers = {
  'Authorization': 'Bearer Token',
  'Content-Type': 'application/json'
}

response = requests.request("GET", url, headers=headers, data=payload)

print(response.text)
```

## Request reference

A sample request containing all the possible parameters

<CodeGroup>

```json cURL expandable wrap
curl --location 'https://api.wokelo.ai/api/enterprise/company/news/?company=tesla&blacklisted=%22https%3A%2F%2Fwww.prnewswire.com%2F%22&limit=100&start_date=2025-01-01&end_date=2026-03-15&category=PARTNERSHIPS&offset=0' \
--header 'Authorization: Bearer Token' \
--header 'Content-Type: application/json'
```

```json Python expandable wrap
import requests
import json

url = "https://api.wokelo.ai/api/enterprise/company/news/?company=tesla&blacklisted=\"https://www.prnewswire.com/\"&limit=100&start_date=2025-01-01&end_date=2026-03-15&category=PARTNERSHIPS&offset=0"

payload = {}
headers = {
  'Authorization': 'Bearer Token',
  'Content-Type': 'application/json'
}

response = requests.request("GET", url, headers=headers, data=payload)

print(response.text)
```

</CodeGroup>

The table below lists all the parameters available to filter the results.

| URL Parameters | Description |
| :-- | :-- |
| `company` | Company permalink or a valid company URL for which news articles need to be fetched |
| `start_date` | Start date for the timeframe to be considered for fetching news details |
| `end_date` | End date for the timeframe to be considered for fetching news details |
| `category` | News Categories to filter the results on. [Download Supported Categories](https://wkemails.blob.core.windows.net/clients/%28%27808dda00-141d-4952-b44c-c50bd41c2ce3%27%2C%29/news_category_names.csv) |
| `limit` | Maximum number of news to return |
| `offset` | Number of records to skip before starting to return results |
| `blacklisted` | Comma-separated list of publisher domains to exclude from results. (include complete domain starting with https://) |

## Best practices

- Set a start\_date / end\_date range to avoid processing large backlogs — 30-day windows work well for regular monitoring runs.
- Use category filters to reduce noise.
- Blacklist low-quality publishers (press release aggregators, forums) to improve signal quality.
- Paginate with limit and offset when processing companies with high news volume (\>500 articles in your date range).
