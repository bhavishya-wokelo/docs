## Quick start

The newsfeed API fetches the latest news articles for a given industry, enriched with AI-generated summaries, event categories, publisher metadata, and article text.

Results are returned synchronously in the response

`POST: /api/enterprise/newsfeed/news/`

NOTE: This is a synchronous API, which means the results will be returned in the HTTP response itself.

Lets start with a simple one.

<CodeGroup>

```json cURL expandable wrap
curl --location 'https://api.wokelo.ai/api/enterprise/newsfeed/news/' \
--header 'Authorization: Bearer Token' \
--header 'Content-Type: application/json' \
--data '{
  "industry": ["Energy, Utilities & Environment", "Industrials & Manufacturing"],
  "breaking_news": true,
  "sentiment":"positive",
  "published_date_after":"2026-04-01",
  "published_date_before":"2026-04-30",
  "limit":100
}'
```

```json Python expandable wrap
import requests
import json

url = "https://api.wokelo.ai/api/enterprise/newsfeed/news/"

payload = json.dumps({
  "industry": [
    "Energy, Utilities & Environment",
    "Industrials & Manufacturing"
  ],
  "breaking_news": True,
  "sentiment": "positive",
  "published_date_after": "2026-04-01",
  "published_date_before": "2026-04-30",
  "limit": 100
})
headers = {
  'Authorization': 'Bearer Token',
  'Content-Type': 'application/json'
}

response = requests.request("POST", url, headers=headers, data=payload)

print(response.text)
```

## Request reference

A sample request containing all the possible parameters

<CodeGroup>

```json cURL expandable wrap
curl --location 'https://api.wokelo.ai/api/enterprise/newsfeed/news/' \
--header 'Authorization: Bearer Token' \
--header 'Content-Type: application/json' \
--data '{
  "industry": ["Energy, Utilities & Environment", "Industrials & Manufacturing"],
  "breaking_news": true,
  "geo":["USA"],
  "sentiment":"positive",
  "published_date_after":"2026-04-01",
  "published_date_before":"2026-04-30",
  "limit":100,
  "offset":2
}'
```

```json Python expandable wrap
import requests
import json

```

</CodeGroup>

The table below lists all the parameters available to filter the results.

| URL Parameters | Description |
| :-- | :-- |
| `industry` | Array of industry names. [View](https://docs.wokelo.ai/industry-names) supported industry names. |
| `geo` | Array of 3 letter ISO country codes |
| `sentiment` | News sentiment- positive, negative, neutral |
| `breaking_news` | Boolean value. To fetch breaking news, set this to true |
| `published_date_after` | Lower bound for date range to filter news articles |
| `published_date_before` | Upper bound for date range to filter news articles |
| `limit` | Maximum number of news to return |
| `offset` | Number of records to skip before starting to return results |
| `blacklisted` | Comma-separated list of publisher domains to exclude from results. (include complete domain starting with https://) |

## Best practices

- Set a `published_date_after`/`published_date_before` range to avoid processing large backlogs — 30-day windows work well for regular monitoring runs.
- Blacklist low-quality publishers (press release aggregators, forums) to improve signal quality.
- Paginate with limit and offset when processing companies with high news volume (\>500 articles in your date range).
