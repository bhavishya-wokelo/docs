## Overview

Starts generating a Company Research report for a specific company.

## Endpoint Details

- **Method:** POST
- **Endpoint:** `/api/workflow_manager/start/`

## Authentication requirements

- Include a valid JWT token in your request header
- If you don't have a token yet, you can get one from the `/auth/token/` endpoint first.

## Request

### Request Parameters

#### Header Parameters

<ParamField header="Authorization" type="string" required>
  JWT token obtained from the Authentication request
</ParamField>


#### Body Parameters

<ParamField body="industry" type="string">
  Company's industry. If not provided, Wokelo will auto-detect the industry
</ParamField>

<ParamField body="permalink" type="string" required>
  Crunchbase permalink of the company
</ParamField>

<ParamField body="website" type="string" required>
  website of the company
</ParamField>

<ParamField body="workflow" type="string" required>
  `company_primer`
</ParamField>

<ParamField body="custom_files" type="array of objects">
  Files to include in the report. Use `fileName` values from the file upload response
</ParamField>

_Note: Either of company's permalink or website is required for the company-based workflows_

## Response

Successful response will include the report\_id of the Company Research report

#### Successful Response Fields

<ResponseField name="report_id" type="integer">
  Report id for the initiated Company Research report
</ResponseField>
